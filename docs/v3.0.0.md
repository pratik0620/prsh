# prsh v3.0.0

Released: 09 July 2026

## What's new

### Raw Terminal Mode

- Switched from `std::getline()` to custom raw terminal input
- Implemented using `termios` — no readline dependency
- `enableRawMode()` disables `ICANON` and `ECHO`, reads one character at a time
- Terminal is restored on exit via `atexit(disableRawMode)`

### Arrow Key Navigation

- Up/Down arrows navigate command history
- Left/Right arrows move cursor within the current line
- Mid-line character insertion works correctly
- Maintains cursor position while editing the command line
- Redraws the current input line using ANSI escape sequences

### Extended Key Support

- Home key — jump to start of line
- End key — jump to end of line
- Delete key — delete character under cursor

### Escape Sequence Parsing

- `readKey()` function parses multi-byte escape sequences
- Returns a typed `Key` enum — clean switch-based dispatch in `readLine()`
- Handles `\033[A/B/C/D` (arrows), `\033[H/F` (Home/End), `\033[3~` (Delete)

### Job Control

- Ctrl+Z (SIGTSTP) suspends the foreground process
- `fg` resumes a suspended job in the foreground
- `bg` resumes a suspended job in the background
- `jobs` lists all tracked background and suspended jobs
- Added a job table to track suspended and background processes

### Signal Handling Improvements

- SIGTSTP handler added alongside existing SIGINT handler
- Child processes reset both SIGINT and SIGTSTP to default before exec
- Foreground jobs can be suspended and resumed using shell job control

## Architecture change

- Added `line_editor.cpp` — owns raw mode input loop and key handling
- Added `terminal.cpp` — owns termios setup and teardown
- Added `jobs.cpp` — owns job state tracking
- `ReadResult` struct with `ReadStatus` enum replaces raw string return from input

## Bug fixes from v2.0.0

- Fixed `atexit(disableRawMode)` registering multiple times — static flag added
- Fixed `CTRL_C` and `ERROR` key cases missing from `readLine` switch
- Fixed terminal staying in raw mode on Ctrl+C at empty prompt

## Known limitations

- No tab completion
- No quoted string support
- No POSIX process groups (`setpgid`, `tcsetpgrp`)
- Foreground job control is implemented without terminal process group management
